# v2.17.0 — A2 Token Tracking + P0 Code Fixes

## A2 Token 使用量追踪 (Feature)

### Backend
- **Token Stats DB**: `data/token_stats.db` with `token_calls` table
  - Fields: ts, date, model, provider, session_id, call_count, tokens_in, tokens_out, duration_ms
- **Data Collector**: `_collect_token_stats()` - parses JSONL session files, counts model calls
  - 5-min scan interval with threading lock
  - 30-day data retention cleanup
  - Model cost estimation (MODEL_COSTS map for deepseek, claude, gpt, qwen pricing)
- **API Endpoint**: `GET /api/tokens?range=7d|30d`
  - Returns: total_calls, total_tokens, total_cost_est, models[], daily[]
  - Each model: calls, tokens_in, tokens_out, duration_ms, cost_est
  - Each daily entry: date, calls, tokens_in, tokens_out

### Frontend
- **initTokenPanel()**: Renders token usage panel between history chart and gauges
  - Summary cards: 调用次数 / Token预估 / 成本预估
  - Horizontal bar chart: model distribution ranked by call count
  - Color coding: green (<20), orange (20-100), red (>100 calls)
  - Daily activity sparkline (7-day bar chart)
  - Empty state: "暂无 Token 使用数据 - 模型被调用后数据将自动出现"

## P0 Code Fixes

### except:pass → logging (22 occurrences)
- All non-BrokenPipeError except blocks now log via `logging.debug()`
- bare `except:` and `except Exception:` patterns now capture exception for logging

### discover_configs caching (P2)
- Added 5-minute TTL cache to `discover_configs()`
- `_discover_cache = {'data': None, 'ts': 0, 'ttl': 300}`
- Reduces file I/O on every collect_all cycle

## Technical Notes
- Token data source: `~/.qclaw/*.jsonl` session files
- Current limitation: modelroute layer doesn't pass actual token counts (records 0)
- Infrastructure is ready; will auto-fill when token data becomes available
- Cost estimates are approximate and model-dependent
